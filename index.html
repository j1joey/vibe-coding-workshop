01 Get Your Token

Takes about 60 seconds.

  1. Go to cf.nebulaone.ai and sign in
  2. Click your profile icon (bottom-left)
  3. Click "Settings"
  4. Select "Access Tokens" from the settings menu
  5. Click "New Token"
  6. Name it (e.g. "Workshop") and click Generate
  7. Copy immediately — you won't see it again
Save your token! Starts with n1pat_. Paste it in the bar at the top of this page.

🔒 Keep It Safe

Your token is powerful. Here's how to use it responsibly.

Your token = your identity
Anything done with your token is done as you. Treat it like a password.
Don't share it
Never paste your token in Slack, Teams, screenshots, or someone else's app.
Revoke after the workshop
Settings → Access Tokens → Revoke. Takes 10 seconds. Do it today.
Don't hardcode tokens
Claude's apps ask for your token at runtime. Never edit the code to embed it.
Watch what agents say
Some agents access company data. Don't share responses with confidential info externally.
Your app stays local
The HTML runs in your browser as you. Don't share generated files outside Cloudforce.
Don't bulk-export data
Build useful tools, not data scrapers. Don't build apps that extract or export company data in bulk.
Client data stays confidential
If an agent returns client names, project details, or NDA content, it stays on your screen. Don't copy it into personal notes or external tools.
Company devices only
Use the workshop app and anything you build on your work computer, not personal devices.

Full guidelines: Security & Data Handling Guidelines | How to Use Personal Access Tokens

05 Troubleshooting

Common issues and what to tell Claude to fix them.

ProblemSolution
"Token exchange failed (401)"PAT expired or revoked. Generate a new one.
"No response received"SSE parsing issue. Tell Claude: "The API sends event and data on separate lines with a blank line between. Don't reset event name on blank lines."
"connection-oauth-expired"Agent needs browser OAuth. Try a different agent.
CORS errorServe locally: python -m http.server 8080
Garbled charactersTell Claude: "Use TextDecoder for base64, not raw atob(). Data is UTF-8."
Claude's code doesn't workNormal! Paste the error back and say "fix this." That's vibe coding.
